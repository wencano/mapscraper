MapScraper Architecture

Leadgen demo · Next.js · PostgreSQL · OpenStreetMap public APIs

Context Containers Pipeline States Data Schedule

Free-map leadgen, built for a crisp demo

Operators define areas and search intent. The server geocodes with Nominatim, discovers POIs via Overpass, upserts leads into Postgres, and surfaces progress in a Notion-styled dashboard.

Single operator talks to MapScraper; MapScraper talks to public OSM APIs and Postgres.

Demo constraint

No Google Maps HTML scraping. Public OSM endpoints only, with polite User-Agent and rate limits.

Why this split

Nominatim resolves addresses → coordinates. Overpass answers “what businesses are near here?”

Persistence

Projects, schedules, scrape progress, and upserted leads live in PostgreSQL.

3. Scrape pipeline

Per location: geocode → Overpass radius query → map tags → upsert. Progress advances after each line.

flowchart LR
  A[Location line] --> B[Geocode]
  B --> C{lat/lon?}
  C -->|yes| D[Overpass ~1.2km]
  C -->|no| E[Skip]
  D --> F[Normalize POIs]
  F --> G[Upsert Lead]
  E --> H[progress++]
  G --> H
  H --> I{more?}
  I -->|yes| A
  I -->|no| J[DONE + nextRunAt]

4. Status state machine

UI badges map 1:1 to `ProjectStatus`. Scraping blocks concurrent starts (409).

stateDiagram-v2
  [*] --> READY
  READY --> SCRAPING: run / cron
  SCRAPING --> DONE: success
  SCRAPING --> ERROR: failure
  DONE --> SCRAPING: re-run
  ERROR --> SCRAPING: retry
  DONE --> READY: edit
  ERROR --> READY: edit

6. Auto-scrape scheduling

`nextRunAt` is computed from schedule enum. Tick endpoint starts due projects that are not already scraping.

sequenceDiagram
  participant Cron
  participant Tick as /api/cron/tick
  participant DB
  participant Engine
  Cron->>Tick: POST
  Tick->>DB: due projects
  Tick->>Engine: start each
  Engine->>DB: upsert leads

Intervals

NONE · EVERY_6H · DAILY · WEEKLY — stored on the project, editable in the settings modal.

Demo wiring

Client can optionally ping tick; or call with curl/cron. Swap later for a real worker.

Upsert semantics

Existing OSM ids refresh `lastSeenAt`; brand-new POIs appear as new table rows.
